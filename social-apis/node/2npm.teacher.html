NPM

Node Package Manager

npm ci look for dependencies in package.json and install when missing

npm update look for dependencies in package.json but it will get the latest

npm outdated compare currently installed modules with latest

node_modules folder stores all the open-source dependencies

Setup

  1. Create a new folder api-visualizer in course-files/src
  2. Create package.json npm init will setup your package.json with a step-by-step wizard
  3. version <major>.<minor>.<patch>. The difference between major versions is a breaking change. Meaning the developer will likely be required to change their code to use the new version. Minor version is safe to consume as there are no breaking changes, likely new features are added. For example the current version of jQuery is v2.1.4, when v2.2.0 is released we can all upgrade to this version without changing our code. Though we should test our web pages to ensure functionality. Patch version will contain bug fixes and security updates. semver (Semantic Versioning)
  4. Choose a license
